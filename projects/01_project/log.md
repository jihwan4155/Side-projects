### 2026-04-02 (Day 3)
- **오늘의 목표:**
  - API 수집 데이터와 SQLite 연동 및 자동 저장 로직(ETL) 구현
  - 다중 키워드 수집을 위한 `keywords` 테이블 연동 및 자동화
- **진행 상황:**
  - 기능의 모듈화: `fetch_and_save_news(keyword)` 함수를 만들어 재사용성 확보
  - 데이터 클리닝 심화: 뉴스 제목 내 불필요한 HTML 태그와 특수문자를 제거
  - 키워드 관리 자동화: 코드에 검색어를 직접 쓰지 않고, DB에서 키워드 목록을 불러와 반복문으로 전체 수집하도록 개선
  - 중복 방지: `INSERT OT IGNORE` 를 통해 이미 저장된 뉴스는 건너뛰고 새로운 뉴스만 저장하는 로직 궇녀
- **오늘의 도전 & 해결 (TIL):**
  - 문제: 뉴스 수집을 반복할 때마다 중복 데이터 처리가 까다로웠음
  - 해결: SQL의 `UNIQUE` 제약 조건과 `IGNORE` 키워드를 활용해 코드 복잡도를 낮추고 데이터 무결성 지킴
- **다음 할 일:**

  - 특정 뉴스를 읽었을 때 `is_read` 상태를 `1`로 바꾸는 업데이트 기능 구현
